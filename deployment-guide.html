🌐 部署选项

您可以选择以下任意一种方式部署游戏：

🚀 选项1：使用GitHub Pages（推荐）

1 创建GitHub账号

访问 GitHub.com 注册账号

2 创建新仓库

点击"New repository"，填写仓库名称（如"kids-racing-game"）

3 上传文件

进入仓库，点击"Upload files"，上传解压后的 webapp 文件夹中的所有文件

4 启用GitHub Pages

进入仓库设置 → Pages → 选择分支为 main → 点击"Save"

5 获取链接

几分钟后，访问生成的GitHub Pages链接（格式：https://用户名.github.io/仓库名）

🌍 选项2：使用Netlify

1 注册Netlify账号

访问 Netlify.com 注册账号

2 拖拽上传

登录后，直接拖拽解压后的 webapp 文件夹到Netlify界面

3 获取链接

上传完成后，Netlify会自动生成一个随机域名链接

💻 选项3：使用Vercel

1 注册Vercel账号

访问 Vercel.com 注册账号

2 导入项目

点击"New Project" → 选择"Import Git Repository"或直接上传文件

3 部署

保持默认设置，点击"Deploy"按钮

4 获取链接

部署完成后，获取生成的Vercel链接

🏢 选项4：自有服务器部署

1 上传文件

使用FTP工具（如FileZilla）将 webapp 文件夹上传到您的服务器

2 配置Web服务器

确保您的Web服务器（Apache/Nginx）正确配置了静态文件服务

3 访问链接

通过您的域名或服务器IP访问游戏（格式：http://您的域名/webapp）

🔧 部署后的配置

域名自定义（可选）

如果您有自己的域名，可以将其绑定到部署的游戏：

  • GitHub Pages： 在仓库设置中添加自定义域名
  • Netlify： 在项目设置 → Domain management中添加域名
  • Vercel： 在项目设置 → Domains中添加域名

HTTPS配置

为了确保游戏可以作为PWA安装，建议启用HTTPS：

  • GitHub Pages、Netlify、Vercel都自动提供HTTPS
  • 自有服务器需要配置SSL证书（可使用Let's Encrypt免费证书）

📝 重要提示

  • 确保 manifest.json 中的 start_url 路径正确
  • Service Worker需要在HTTPS环境下才能正常工作
  • 如果修改了文件名或路径，需要更新相关引用
  • 定期检查并更新游戏版本以获得最佳体验

🚨 常见问题排查

Q: 游戏无法作为PWA安装？
A: 检查是否启用了HTTPS，以及manifest.json文件是否正确配置

Q: 游戏资源加载缓慢？
A: 可以考虑使用CDN加速静态资源

Q: 安装后无法离线使用？
A: 检查Service Worker是否正确注册，以及缓存资源列表是否完整
